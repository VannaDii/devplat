# Security Policy

## Reporting a Vulnerability

Do not report security issues in public issues, pull requests, or discussions.

Use GitHub private vulnerability reporting when available for this repository. If private reporting is unavailable, contact the maintainers privately through GitHub.

Include:

- affected package or path
- impact
- reproduction details
- suggested mitigation if known

## Response Expectations

Maintainers will triage reports, assess severity, and coordinate remediation before public disclosure.

## Scope

Security-sensitive areas include:

- policy enforcement
- OpenClaw and Discord control surfaces
- GitHub automation
- storage and audit persistence
- execution and gate orchestration
